# 🎵 Music Store Data Analysis using SQL

## 📌 Project Overview

This project focuses on analyzing a music store database using SQL to extract meaningful business insights. The analysis covers customer behavior, sales trends, and product performance by querying relational data across multiple tables.
---
## 🎯 Objectives
* Identify top customers based on spending
* Analyze sales performance by city and country
* Find best-selling tracks and most popular artists
* Evaluate invoice trends and revenue distribution
* Detect customers spending above average
---
## 🗂️ Database Schema

The project uses a relational database with the following key tables:
* Customer
* Invoice
* Invoice_Line
* Track
* Album
* Artist
* Genre
* Employee
---
## 🛠️ Tools & Technologies

* SQL (MySQL)
* Relational Database Management System (RDBMS)

---

## 🔍 Key Analysis Performed

### 1. Senior Most Employee

* Identified employees with the highest job level using sorting techniques.

### 2. Country-wise Invoice Analysis

* Determined which countries generate the most invoices.

### 3. Top Invoice Values

* Extracted top 3 highest invoice totals.

### 4. City with Highest Revenue

* Analyzed total invoice amount by city to find top-performing locations.

### 5. Best Customer

* Identified customers contributing the highest revenue.

### 6. Rock Music Listeners

* Retrieved customer details who listen to rock music using joins across multiple tables.

### 7. Top Artist by Track Count

* Found artists with the highest number of tracks in the database.

### 8. Customers Spending Above Average

* Used subqueries to filter customers spending more than the average.

### 9. Best-Selling Tracks

* Identified tracks with the highest purchase frequency.

---

## 📊 Key Insights

* A small group of customers contributes significantly to total revenue.
* Certain cities and countries dominate sales performance.
* Popular artists and tracks drive the majority of purchases.
* Customer spending behavior varies significantly across regions.
